    正则表达式

    pioneer 9/23/2022

    # 正则表达式

        1. 匹配字符
       匹配字符A, /A/.test("JAVA") => true, /A/.test("JBVB") => false
       匹配以A开头, /^A/.test("ABC") => true, /^A/.test("CBA") => false
       匹配以A结尾, /A$/.test("ABC") => false, /A$/.test("CBA") => true
       匹配A或B, 把对应字符放入中括号里, /[AB]/.test("BYTE") => true
       匹配字符串ABC或XYZ, /ABC|XYZ/.test("AABBXYZ") => true

       匹配(前瞻), exp1(?=exp2): 匹配exp2前面的exp1, 匹配结果不包含exp2
            /java(?=script)/.test("javascript") => true
            /java(?=script)/.test("javasscript") => false
            /java(?=script)/.exec("javascript") => [
          "java"
       ]
       匹配(后顾), (?<=exp2)exp1: 匹配exp2后面的exp1, 匹配结果不包含exp2
                    /(?<=java)ee/.test("javaee") => true
                    /(?<=java)ee/.exec("javaee") => [
          "ee"
       ]

    2. 匹配不是
       取反, 只要不是这些就能匹配, 在中括号里面加上元字符^, 只要字符满足有不是这个集合里的字符, 都能被匹配上
       /[^a]/.test("aaa") => false
       /[^a]/.test("abc") => true
       匹配不是以a开头的:
       /^[^a]/.test("javascript") => true
       /^[^a]/.test("abc") => false
       匹配不以a结尾的:
       /[^a]$/.test("javascript") => true
       /[^a]$/.test("cba") => false
       不匹配字符a,b,c, 可以把对应的字符都放入括号里:
       /[^abc]/.test("abccba") => false
       /[^abc]/.test("abccbad") => true

       匹配后面不是(负前瞻)
       exp1(?!exp2): 匹配后面不是exp2的exp1, 匹配结果不包含exp2
       匹配后面不是script的java
       /java(?!script)/.test('javaee') => true
       /java(?!script)/.exec('javaee') => [
          "java"
       ]
       匹配前面不是(负后顾)
       (?<!exp2)exp1: 匹配前面不是exp2的exp1, 匹配结果不包含exp2
       匹配前面不是java的script
       /(?<!java)script/.test('javascript') => false
       /(?<!java)script/.exec('javasscript') => [
          "script"
       ]

       不匹配包含abc的字符串
       A. 位置后面不能为abc, 使用负前瞻匹配位置: /(?!abc)/
       B. 开始到结束每个位置都要覆盖到, 开始结束标记: /^(?!abc)$/
       C. 这个位置后面可以是其他字符, /^(?!abc)\w$/
       D. 满足上面情况的位置: 可以出现多次, 用 + 来表示数量

       /^((?!abc)\w)+$/.test("cbacbac") => true
       /^((?!abc)\w)+$/.test("cbacbabc") => false

    3. 匹配多少次?
       *      0次或多次
       +      1次或多次
       ?      0次或1次
       {m}    匹配m次
       {m,n}  最低匹配m次, 最多匹配n次, m<n
       {m,}   最低匹配m次

       .  匹配除换行任意字符
       \d 匹配任意数字
       \D 匹配任意非数字
       \s 匹配任意空白字符
       \S 匹配任意非空白字符
       \n 匹配换行
       \w 匹配字母数字下划线, [A-Za-z0-9_]

       i 忽略大小写;
       m 多行匹配;
       g 全局匹配;

       匹配特殊字符要加反斜杆 \
       JS中特殊字符有 ^ $ \ . * + ? () [] {} |

       正则默认贪婪匹配, 符合条件会一直匹配, 想阻止贪婪匹配, 可以在长度规则后面加一个 ?
       /\d{2,}/.exec("123456789") => [
         "123456789"
       ]
       /\d{2,}?/.exec("123456789") => [
         "12"
       ]

       使用分组
       /(\d)\1(\d)\2/.exec("1234445556789") => [
         "4455",
         "4",
         "5"
       ]

       默认分组可以被捕获, \1, \2是正则表达式内部捕获的分组, 想去外部捕获分组匹配的数据可以
       使用RegExp.$1-$9来获取。只要正则匹配了就会有。可以使用test、exec或者str的replace方
       法来获取$1-$9。
       使用test:
       /([a-z]{2})(\d{2})/.test("xyz123") => true

       使用replace:
       "xyz123".replace( /([a-z]{2})(\d{2})/, "$2$1") => x12yz3

       分组不捕获
       不想捕获分组, 只需要在分组内加上?:就行
       /([a-z]{2})(?:\d{2})/.test("xyz123") => true
       RegExp.$1 返回第一个分组表达式匹配到的内容 yz
       RegExp.$2 分组未被捕获, 返回空字符串
